[TCC] Contribuição ao projeto open-source plugdata

Este trabalho de conclusão de curso é de autoria de Gabriel Haruo Hanai Takeuchi, sob orientação do Prof. Dr. Marcelo Gomes de Queiroz, e será desenvolvido no ano de 2026.

Resumo da Proposta

O trabalho propõe a contribuição direta ao projeto open-source plugdata, uma interface moderna para o ambiente de programação visual Pure Data (Pd). Com mais de duas mil estrelas e vinte contribuidores no Github, plugdata é uma ferramenta globalmente reconhecida e importante para a comunidade de processamento de sinais digitais de áudio e também para a comunidade artística de música eletrônica.

O foco central será o aprimoramento da ferramenta por meio da resolução de issues listadas no repositório oficial, abrangendo desde correções de pequenos bugs até implementações de novas funcionalidades utilizando o framework JUCE em C++.

Será visada a aplicação de técnicas de engenharia de software e processamentos de sinais em um ambiente de produção real, além de documentação contínua dos ciclos de desenvolvimento e comunicação constante com os mantenedores.

Fundamentação Teórica

O projeto se fundamenta na pesquisa sobre Processamento de Sinais Digitais aplicado a sistemas de processamento de áudio em tempo real. As principais ferramentas de desenvolvimento serão a linguagem de programação C++ e o framework JUCE, padrão da indústria para o desenvolvimento de software musical multiplataforma. O motor de áudio do plugdata é oriundo do Pure Data (Pd), um ambiente de programação visual que permite a manipulação de fluxos de dados multimídia.

Metodologia

A metodologia de desenvolvimento seguirá o modelo open-source via GitHub, estruturada nas seguintes etapas:

  • Exploração e Build: Configuração do ambiente de desenvolvimento e análise da integração entre a interface gráfica em JUCE e o kernel do Pd.
  • Escolha de Issues: Seleção de issues no repositório oficial baseada em prioridade e escopo.
  • Ciclos de Desenvolvimento: Implementação de testes, correções e funcionalidades seguindo fluxos de trabalho baseados em Git Flow e princípios de metodologias de desenvolvimento ágil.
  • Code Review: Iteração com os mantenedores do projeto para ajuste e aprovação das contribuições.
